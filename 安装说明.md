## 🛠️ 本地编译与打包安装指南

如果您需要自己修改代码并编译成原生安装包，请按以下步骤操作：

### 第一步：配置本地开发环境
项目目前使用的是配置在 `E:\flutter` 的 SDK。
1. 打开 PowerShell 终端，进入项目根目录：
   ```powershell
   cd e:\AI-based-project\love-app
   ```
2. 设置环境变量并获取依赖：
   ```powershell
   $env:PATH = "E:\flutter\bin;$env:PATH"
   flutter pub get
   ```
3. 验证代码静态健康：
   ```powershell
   flutter analyze
   ```

---

### 第二步：Android 客户端打包与安装 (APK)
1. 在终端中运行安卓 Release 编译命令：
   ```powershell
   $env:PATH = "E:\flutter\bin;$env:PATH"
   flutter build apk --release
   ```
2. 编译完成后，安装包位于以下路径：
   👉 `e:\AI-based-project\love-app\build\app\outputs\flutter-apk\app-release.apk`
3. 将此 APK 文件发送到您的安卓手机上，点击安装即可。

---

## 第四步：双人配对与使用说明

当两台手机都成功“安装”（或打开网页版）并进入 App 后：

1. **注册账号**：
   - 两人分别使用自己的手机号或邮箱注册一个账号。
2. **发起配对**：
   - 登录后，App 将自动进入**配对界面**。
   - 其中一人输入另一人的注册账号（手机号/邮箱），点击**“发起配对”**。
   - 另一人刷新或重新打开 App，接受配对申请。
3. **开始同步使用**：
   - 配对成功后，App 会自动进入首页。您们可以开始体验跳动爱心、相册照片墙、纪念日日历、心愿时光轴！所有数据均会自动上传云端同步。

## 🛠️ 常见问题排查 (FAQ)

#### Q1: 数据能正常保存和同步吗？
* **解答**：可以。我们已经成功配置了 TapTap TDS 服务，所有数据在两人注册配对后，都是实时进行云端同步的。

#### Q2: 提示“The argument type 'File' can't be assigned...”编译报错？
* **解答**：这是由于 Web 平台和手机端平台对 `File` 类型的解释冲突。我们已经通过 `lib/widgets/local_image_helper.dart` 条件载入机制修复了此问题。如果遇到该报错，请确保运行了最新版本的代码。
