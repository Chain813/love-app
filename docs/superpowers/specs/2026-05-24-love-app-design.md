# 虫米 App 设计文档

> 情侣恋爱记录 App，支持 Android + iOS

## 1. 项目概述

### 1.1 产品定位

虫米是一款面向情侣的恋爱记录 App，帮助情侣记录恋爱点滴、管理纪念日、共享相册、互动小游戏等功能。

### 1.2 目标用户

- 中国大陆用户
- 所有年龄段情侣
- 无开发经验的个人开发者

### 1.3 核心价值

- 记录恋爱美好时光
- 增进情侣互动
- 简洁优雅的 Apple 风格 UI

---

## 2. 技术架构

### 2.1 技术选型

| 项目 | 选型 | 说明 |
|------|------|------|
| 前端框架 | Flutter | 一套代码双端运行，性能接近原生 |
| 后端服务 | LeanCloud | 国内 BaaS，免费额度充足 |
| 本地缓存 | Hive | Flutter 本地存储，提升读取速度 |
| 登录方式 | 微信登录 | 国内主流，用户无需注册 |
| 微信分享 | fluwx 插件 | 支持微信链接分享 |

## 3. UI 设计规范

### 3.1 设计原则（参考"不背单词"风格）

| 原则 | 说明 |
|------|------|
| 图标优先 | 核心功能以图标为主，文字为辅，界面更简洁 |
| 大留白 | 大量留白，信息层次清晰，呼吸感强 |
| 圆润 | 圆角卡片、圆角按钮、柔和阴影 |
| 字体 | 使用系统字体，大标题加粗，正文简洁 |
| 动效 | 流畅的过渡动画，符合物理直觉 |
| 色彩 | 主题色柔和，不刺眼，背景纯净 |
| 插画 | 使用精致的小插画/图标增强视觉表现力 |

## 4. 功能模块设计

### 4.1 用户系统

#### 4.1.1 核心规则

| 规则 | 说明 |
|------|------|
| 一对一绑定 | 每个用户只能绑定一位伴侣，绑定后不可更改 |
| 配对方式 | 一方生成邀请码，另一方输入邀请码完成配对 |
| 分手处理 | 需注销账号，所有数据清除，重新注册后可绑定新伴侣 |
| 账号注销 | 注销前需二次确认，数据不可恢复 |

#### 4.1.2 用户状态流转

```
┌──────────┐    微信登录    ┌──────────┐   输入邀请码   ┌──────────┐
│  未注册   │ ──────────── → │  已注册   │ ──────────── → │  已配对   │
│          │               │  未配对   │               │          │
└──────────┘               └──────────┘               └──────────┘
                              ↑                         │
                              │      注销账号            │
                              └─────────────────────────┘
                              (数据清除，重新开始)
```

#### 4.1.3 数据库设计

**用户表 (_User)**

```json
{
  "objectId": "LeanCloud 自动生成",
  "wechat_openid": "微信 OpenID",
  "nickname": "昵称",
  "avatar": "头像 URL",
  "couple_id": "情侣 ID（未配对时为 null）",
  "partner_id": "伴侣 ID（未配对时为 null）",
  "invite_code": "邀请码（未配对时生成）",
  "status": "single | paired",
  "createdAt": "注册时间",
  "pairedAt": "配对时间"
}
```

**情侣表 (Couple)**

```json
{
  "objectId": "LeanCloud 自动生成",
  "user1_id": "用户1 ID",
  "user2_id": "用户2 ID",
  "anniversary_date": "在一起纪念日",
  "status": "active | dissolved",
  "createdAt": "配对时间"
}
```

#### 4.1.4 分手注销流程

```
用户点击"注销账号"
    ↓
弹出警告："注销后所有数据将永久删除，不可恢复"
    ↓
用户确认
    ↓
云函数执行：
1. 删除情侣关系
2. 删除所有共同数据（日记、相册、心愿等）
3. 删除用户账号
    ↓
跳转到登录页面
```

#### 4.6.4 小游戏

**房间系统流程**

```
用户A 创建房间
    ↓
生成房间码（6位数字）
    ↓
用户A 点击"邀请对方" → 调用微信分享（分享房间码）
    ↓
用户B 收到消息 → 打开 App → 输入房间码加入
    ↓
双方都点击"准备"按钮
    ↓
3秒倒计时 → 游戏开始
    ↓
游戏进行中...
    ↓
游戏结束 → 显示结果 → 返回房间
```

**说明**：由于用户没有自己的域名，邀请方式采用**分享房间码**而非链接。用户B收到房间码后，在App内输入即可加入房间。

## 8. 注意事项

### 8.1 微信登录

- 需要在微信开放平台注册应用
- 需要申请微信登录权限
- 审核周期约 1-2 周

### 8.2 LeanCloud

- 免费额度足够初期使用
- 超出免费额度后需要付费
- 建议在 App 中加入使用量监控

### 8.3 照片存储

- 上传前压缩照片，节省存储空间
- 生成缩略图，加快列表加载
- 定期清理无用文件

### 8.4 数据安全

- 敏感数据加密存储
- 定期备份数据
- 注销账号时彻底清除数据
